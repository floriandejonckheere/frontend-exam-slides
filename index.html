Ruby

  • Designed by Yukihiro Matsumoto (Matz) in 1995
  • Dynamic (interpreted) language, like JavaScript or Python
  • Dynamic typing
  • Loosely based on Perl

Ruby

  • Truly object oriented
  • No primitives
  • Reflection is builtin
  • Operators are methods
  • Dynamic dispatch

Ruby

  • JavaScript: competition between two companies
  • PHP: community effort
  • Ruby: vision of a single person
  • Rails: vision of another single person

Why Ruby

  • Write more understandable code in less lines
  • Very permissive license
  • Japanese design aesthetics
    • Principle of least surprise
    • Principle of succinctness

Principle of least surprise

What class is an object?
object.class

Length of an array?
arr.length, arr.size or arr.count

Collection operations?
arr1 + arr2, arr1 - arr2

Duck typing

If it looks like a duck, swims like a duck, and quacks like a duck,
then it is a duck.
