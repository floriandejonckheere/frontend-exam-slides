Ruby

  • Designed by Yukihiro Matsumoto (Matz) in 1995
  • Dynamic (interpreted) language, like JavaScript or Python
  • Dynamic typing
  • Loosely based on Perl

Ruby

  • Truly object oriented
  • No primitives
  • Operators are methods: [], +, -, <<
  • Double dispatch
  • Metaprogramming

Why Ruby

  • Write more understandable code in less lines
  • Very permissive license
  • Japanese design aesthetics
    • Principle of least surprise
    • Principle of succinctness

Principle of least surprise

What class is an object?
object.class

Length of an array?
arr.length, arr.size or arr.count

Collection operations?
arr1 + arr2, arr1 - arr2

Duck typing

If it looks like a duck, swims like a duck, and quacks like a duck,
then it is a duck.

RubyGems

  • Package manager like NPM or pip
  • Enable modular applications

Ruby on Rails

  • Server-side web app framework
  • Written in Ruby
  • Optimized for programmer happiness and productivity
  • MVC-pattern
  • Convention over Configuration
  • Don't Repeat Yourself (DRY)

Ruby on Rails

  • Based on REST principles
  • Batteries included
    • JSON, XML, HTML response rendering
    • Database infrastructure
    • Email infrastructure
    • API-only mode
    • Behaviour Driven Development via RSpec

Ruby on Rails

  • Scaffolding/boilerplate code
    • Make it work first, then expand it
    • Prototyping

Models

  • Database wrapping instead of Database mapping
  • Encapsulates database and domain logic
  • Based on Martin Fowler's ActiveRecord pattern
  • Still easy to drop to SQL
      
class Blog < ActiveRecord::Base
  has_one :author, :class_name => 'User'
  has_many :posts
  has_many :categories
end

Migrations

  • Database-independent schema definitions
  • Incrementally build your database
      
class CreateUsers < ActiveRecord::Migration[5.0]
  def change
    create_table :users do |t|
      t.string :name
      t.string :email, :null => false

      t.timestamps
    end
  end
end

Controllers

Disadvantages

  • Way slower than Java
  • No real concurrency due to Global Interpreter Lock (MRI)
  • Small-ish market share
  • Easy to program != good programming (see PHP)

Future

  • JavaScript does not completely suck anymore
  • Shift to SPA
  • Prototyping

Conclusion

  • Ruby is awesome
  • Used more and more to teach OOP and MVC

  • Essential tool for a polygot programmer
